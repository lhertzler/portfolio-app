Phase 1 – Global Layout, Providers & State

Goals: Root layout, global providers, UI + player stores wired up.

1.1 Create state stores

File: src/store/ui-store.ts
	•	State:
	•	themeMode: "light" | "dark" | "studio"
	•	accentColor: string
	•	motionPreference: "full" | "reduced"
	•	inspectMode: boolean
	•	isThemePanelOpen: boolean
	•	isTerminalOpen: boolean
	•	Actions:
	•	setThemeMode(mode)
	•	setAccentColor(color)
	•	setMotionPreference(pref)
	•	toggleInspectMode()
	•	openThemePanel(), closeThemePanel()
	•	openTerminal(), closeTerminal()
	•	Add a simple persist via localStorage (client-side only).

File: src/store/player-store.ts
	•	Types:
	•	Track type with fields: id, title, artist?, src, bpm?, duration?, coverArt?.
	•	State:
	•	queue: Track[]
	•	currentTrackId: string | null
	•	isPlaying: boolean
	•	positionSeconds: number
	•	Actions:
	•	setQueue(tracks: Track[])
	•	playTrack(id: string)
	•	pause(), resume()
	•	next(), prev()
	•	seek(seconds: number)

1.2 Global data files

File: src/lib/tracks.ts
	•	Export a small array of Track objects for your personal songs.

File: src/lib/projects.ts
	•	Export initial project data with:
	•	slug, title, role, summary, tags, tech, featured: boolean, description, caseStudySections.

File: src/lib/command-palette-data.ts
	•	Export an array of commands:
	•	{ id, type: "page" | "project" | "action", label, shortcut?, href?, actionKey? }
	•	actionKey can be used to route into specific handlers (e.g., toggle-theme, open-terminal, play-track-prisoner).

	•	Hide on touch devices.
	•	Respect reduced motion preference (fall back to default cursor if needed).

Phase 4 – Persistent Player + Waveform Visualizer

Goal: Global audio player with a simple visualizer.

4.1 Audio player UI

File: src/components/player/audio-player.tsx
	•	Position: fixed bottom full-width, centered content.
	•	Elements:
	•	Current track title + artist.
	•	Previous / Play-Pause / Next buttons.
	•	Scrubber (progress bar).
	•	Minimal time elapsed / remaining.
	•	<WaveformVisualizer /> embedded.
	•	Hooks into:
	•	player-store for state/actions.
	•	HTML <audio> element (ref) for playback.
	•	useEffect to同步 audio element and store state.

4.2 Waveform visualizer

File: src/components/player/waveform-visualizer.tsx
	•	Use Web Audio API:
	•	Take audioElement as prop or create internal AudioContext.
	•	Create AnalyserNode.
	•	In requestAnimationFrame loop, get frequency/time-domain data.
	•	Render:
	•	A row of n bars (e.g., 32–64 divs) whose heights are driven by analyser data.
	•	Keep styling subtle (thin bars, soft rounding).

4.3 Track helpers

File: src/components/player/track-list.tsx
	•	Accepts array of tracks.
	•	Renders:
	•	Track name, optional “play” button.
	•	On click:
	•	Calls playTrack(track.id) from player-store.

Use this small component in Home / About / Lab later.

Phase 7 – Dev Tools: Inspect Mode & Mini Terminal

7.1 Inspect Mode

File: src/components/devtools/inspector-toggle.tsx
	•	Small icon button:
	•	Calls toggleInspectMode() from ui-store.

File: src/components/devtools/inspector-overlay.tsx
	•	Only renders when inspectMode === true.
	•	Behavior:
	•	Full-screen pointer-events-none overlay with pointer-events-auto on highlight boxes.
	•	On mousemove, find the element under cursor that has data-component attribute.
	•	Draw a highlight box and label near it with:
	•	data-component
	•	optional data-file if present.
	•	Implementation hint:
	•	Use document.elementFromPoint(x, y) and walk up parentElement until data-component found.

7.2 Mini Terminal

File: src/components/devtools/mini-terminal.tsx
	•	Only shows when isTerminalOpen === true.
	•	UI:
	•	Panel that slides up from bottom (use motion.div or Sheet-style).
	•	Command history list + single input line.
	•	Basic command handler:
	•	help – lists commands.
	•	whoami – prints short bio string.
	•	skills – prints core stacks.
	•	projects – lists project slugs.
	•	open <slug> – router.push('/portfolio/' + slug).
	•	theme <light|dark|studio> – set theme.
	•	play <trackId> – call playTrack.
	•	clear – clears history.
	•	Store command definitions in a small local map inside terminal component for now.

Phase 8 – Motion, Transitions & Polish

Once MVP is working:
	1.	Smart page transitions
	•	Wrap route content in AnimatePresence & motion.div at a shared layout level.
	•	Use simple fade/slide transitions between pages.
	•	Add shared layoutId for project thumbnail → case study hero.
	2.	Reduced motion support
	•	If motionPreference === "reduced", disable complex animations and use instant transitions.
	3.	Refine custom cursor
	•	Different styles for:
	•	Links (data-cursor="link")
	•	Project cards (data-cursor="view")
	•	Terminal icon (data-cursor="hack")
	4.	Polish theming
	•	Tune color tokens for Studio mode (music-y, but subtle).
	•	Ensure good contrast + accessibility.
	5.	Copy & content
	•	Fill in real case study content (esp. SoundVent).
